# Contributing

Contributions are welcome.

Good contributions include:

- new tool-specific instruction files
- measured before/after examples
- prompt templates
- workflow patterns
- context hygiene techniques
- model routing examples
- agent memory compression tips

## Style

- Keep content practical.
- Avoid exaggerated token-saving claims.
- Prefer examples over theory.
- Do not copy proprietary tool prompts or private company instructions.
- Do not sacrifice accuracy for extreme brevity.

## Pull Request Checklist

- [ ] The contribution is practical.
- [ ] The wording is concise.
- [ ] No confidential content is included.
- [ ] Any claims are realistic or clearly marked as examples.
