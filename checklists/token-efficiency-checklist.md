# Token Efficiency Checklist

Use this before starting an AI coding or debugging session.

## Before Prompting

- [ ] I know the exact outcome I want.
- [ ] I removed unrelated context.
- [ ] I included exact errors instead of full logs.
- [ ] I listed relevant files only.
- [ ] I removed duplicated output.
- [ ] I selected the right model for the task.

## During The Session

- [ ] Ask the AI to use token-efficient mode.
- [ ] Ask it to read only relevant files.
- [ ] Ask it to summarise noisy outputs.
- [ ] Stop broad scans if the task is narrow.
- [ ] Start a new chat if the topic changes.

## For Coding Agents

- [ ] Use `AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, or tool-specific instructions.
- [ ] Keep persistent instructions short.
- [ ] Keep project memory clean.
- [ ] Use targeted tests.
- [ ] Ask for concise final summaries.

## After The Task

- [ ] Save only durable project facts.
- [ ] Remove stale memory or instructions.
- [ ] Record token-saving lessons if useful.
- [ ] Do not carry irrelevant context into the next task.
